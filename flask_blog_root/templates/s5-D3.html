What is D3?





  • In short Data Driven Documents.
  • One of the latest and most popular technologies to be created in a long line of data visualization software.
  • Written in and write in pure JS

    Why D3?





  • D3.JS stands out because its an incredibly flexible library built upon web standards, the greatest communcation platform we have thus far.
  • This makes our mission of converting data into easily consummable information and thus hopefully knowledge.
  • Unlimited potential: The ways that the library is being used is still growing in all sorts of ways. Big community, used by "top" orgs, etc... All the makings of a truly lasting open source project.

    How D3?





  • With the implementation of CSS3 selectors and then HTML5 standards including the SVG element it was made possible to write a Js library that could drive shapes and drawings on our html documents with all sorts of data.

    Quick intro to SVG

  • Scalable Vector Graphics.
    MDN
  • Scalable: No need to think much in pixels, rather think in coordinates.
  • Vector: A one-dimensional array
  • Graphics: images defined by geometric primitives as opposed to bitmaps
  • Circle Example:
    • <circle cx="200" cy="200" r="50" fill="red" stroke="blue" stroke-width="10"/>
  • The two important style rules are fill and stroke. Both rules accept either a hex color or a regular CSS color
  • other shapes we get for free:
  • rect, ellipse, line, polyline, polygon, path

    Why D3 sucks

  • Other tools will certainly supercede it, like everything, just a matter of time.
  • All "organized" information has fallacies, even raw data and experiments often have terrible fallacies. Don't dress a pig in J. Lo's grammy dress
  • Have to do lots of extra and tedious things to be able to manipulate your data viz for mobile
  • D3 prides itself on being built upon web standards but we all should know what that means.
  • In this case the compatability range is IE9+

    Lots of Data Viz Tools: Python to the web, maybe with some JS

  • Montalenti says: * d3 bespoke viz: hardest, most flexible * nvd3 chart models: slightly easier, still very flexible * vincent/vega: easiest, relatively inflexible
  • Vega describe data visualizations in a JSON format, and generate interactive views using either HTML5 Canvas or SVG
  • Vincent is a single author building on top of Vega, Vincent is a Python library that “humanizes” vega
  • MatPlotLib is very well established in the python community, or at least scipy / pydata
  • mpld3: A D3 Viewer for Matplotlib - more of a jacky solution, not a ground up project like Bokeh and Plotly
  • Bokeh is one of several open-source components of the broader technical vision of Continuum Analytics. Traditional 3-clause BSD License and developed in part by funding from DARPA‘s XDATA program
  • considered orthogonal to D3.js
  • Bokeh is merely producing JSON objects for BokehJS
  • Plot.ly plotting library is also awesome and "good github page that "puts more of the functionality of R in the hands of a python coder" to use on the web.
  • Tangible is a Python library to convert data into tangible 3D models. It generates code for different backends like OpenSCAD or ImplicitSCAD. It is inspired by projects like OpenSCAD and d3.js.
  • iPython and Pandas to d3py
  • On and on... but these are a good reference point
